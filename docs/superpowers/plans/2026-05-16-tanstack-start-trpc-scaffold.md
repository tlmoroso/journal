# Tanstack Start + tRPC Scaffold Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Scaffold a working Tanstack Start app for the `journal` repo with tRPC v11, the Tanstack ecosystem (Router, Query, Table), pnpm catalog-based version management, and ready-to-use dev scripts.

**Architecture:** Single-package Tanstack Start app (post-Vinxi, Vite-native via `tanstackStart()` plugin). tRPC server lives under `src/server/trpc/` and is mounted on `src/routes/api/trpc/$.ts`. Client uses `@trpc/tanstack-react-query` to bridge tRPC and Tanstack Query. Versions are managed in a single `default` pnpm catalog in `pnpm-workspace.yaml`.

**Tech Stack:** Tanstack Start (Vite + Nitro/h3), Tanstack Router/Query/Table, tRPC v11, Zod 4, TypeScript 5 (strict), ESLint 9 flat config, Prettier 3, Vitest, Node 24 LTS, pnpm 9.5+.

**Design Spec:** [`docs/superpowers/specs/2026-05-16-tanstack-start-trpc-scaffold-design.md`](../specs/2026-05-16-tanstack-start-trpc-scaffold-design.md)

import { fetchRequestHandler } from '@trpc/server/adapters/fetch';
import { appRouter } from '~/server/trpc/routers/_app';
import { createContext } from '~/server/trpc/context';

function handler({ request }: { request: Request }) {

    router: appRouter,
    createContext: () => createContext({ headers: request.headers }),
